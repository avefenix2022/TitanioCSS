Marcado HTML5

El desarrollo web requiere de un amplio conocimiento cognitivo de varias materias muy atomizada de las que hay que disponer del conocimiento, habilidad y experiencia en su uso para un correcto suso.

HTML5 presenta un nuevo escenario que intenta imponer una mayor disciplina en el maquetador web para que utilice las etiquetas correctas en cada momento para marcar semánticamente el contenido que encierran, y que además en este sentido utilice las nuevas etiquetas de HTML5 que definen con mayor claridad su contenido y estandarizan una definición de contenido que permitirá a navegadores o dispositivos que muestren contenido web reconocer partes del contenido y actuar ante él. Por otro lado HTML5 busca depender solo de la capacidad del navegador para mostrar video, sonido, gráficos y animaciones sin depender de terceros. Que sea la propia tecnología del navegador (dentro de un estándar) la que aporte la funcionalidad.

Sintaxis

HTML5 no es una evolución del XHTML, por ese motivo ya no tenemos las reglas estrictas de sintaxis del XHTML.

Elementos HTML, HEAD y BODY

No es obligatorio incluir las etiquetas html, head y body.

Elementos con etiquetas de cierre facultativa

No es necesario incluir el cierre en ciertas etiquetas como <p>, sería válido:

            
<p>Mi párrafo sin etiqueta de cierre

Elementos con etiqueta única de apertura

Con los elementos img, br, etc no es necesitan indicar el cierre de las etiquetas ( />):

   
<br> en lugar de <br />
<img src="imagen.jpg"> en lugar de <img src="imagen.jpg" />

Mayúsculas y minúsculas

HTML5 acepta mayúsculas y minúsculas en la sintaxis y en los atributos:

            
<SCRIPT SRC="miScript.js">
<script src="MIscript.js">
<ScRiPt SrC="MiScript.js">

Comillas

Podemos utilizar las tres sintaxis siguientes:

<meta charset="utf-8"> Comillas dobles
<meta charset='utf-8'> Comillas simples
<meta charset=utf-8>   Sin comillas

Válida la sintaxis con comillas dobles, comillas simples y sin comillas.

Si un atributo tiene con varios valores, es obligatorio el uso de comillas dobles:

     
<div class="estilo1 estilo2"> Con varios valores comillas dobles

Atributos boleanos

En HTML5 la simple presencia del atributo es suficiente, asi frente a:
<input type="checkbox" checked="checked" name="checkbox">,
ahora es válido: 

  
<input type="checkbox" checked name="checkbox">

Sintaxis recomendada

Se recomienda que se respete la sintaxis XHTML:

  • incluir todas las etiquetas estructurales del documento
  • cerrar todas las etiquetas
  • cerrar con /> las etiquetas que se autocierran
  • usar solamente minúsculas
  • usar siempre comillas y mejor dobles
  • para aumentar la legibilidad sangrar el código (4 espacios)

Etiquetas de estructura de un documento HTML5

<!DOCTYPE html>

Definición del tipo de documento para HTML5

  • Debe de ser la primera línea.
  • No debe de haber espacios en blanco antes de ella.

<html>

Elemento raiz de un documento HTML.

El atributo lang define el idioma que se utiliza para el contenido del documento.

            
<html lang="es">

<head>

Cabecera. Contiene información sobre el documento que no se mostrará en la ventana del navegador.

<body>

Cuerpo. Contiene la parte visible que se mostrará en la ventana del navegador.

<title>

Títulos únicos y precisos para cada página. Los títulos de las páginas son un aspecto importante de la optimización para motores de búsqueda.

Una etiqueta title indica tanto a los usuarios como a los motores de búsqueda el tema sobre el que trata una página

El contenido de la etiqueta title, por lo general, aparece en la primera línea de los resultados. Esto puede ayudar a los usuarios a reconocer si la página es relevante para su búsqueda.

Se debe de evitar:

  • un título que no tenga ninguna relación con el contenido de la página
  • el uso de títulos predeterminados o demasiado genéricos como “Sin título” o “Página nueva”
  • el uso de una sola etiqueta title para todas las páginas de tu sitio, o para muchas de ellas
  • títulos muy largos que no sean útiles para los usuarios
  • rellenar las etiquetas title con palabras clave innecesarias

<meta>

Metadatos. Los metadatos contienen información sobre la página. Incluye información sobre estilos, scripts y datos que ayudan al navegador a usar y generar la página. Los metadatos de estilos y scripts pueden estar definidos en la página o estar enlazados a otro fichero que contiene la información. Las etiquetas meta se situan en el head.

     
<meta charset="UTF-8">
<meta name="author"      content="Francisco Marinas">    
<meta name="keywords"    content="HTML, CSS">
<meta name="description" content="Lenguaje de marcado semántico HTML5">
<meta name="viewport"    content="width=device-width, initial-scale=1">  
<meta http-equiv="X-UA-Compatible" content="IE=edge"> 
Si utilizas el doctype de HTML5, la página ya se mostrará automáticamente 
en el modo de compatibilidad de Internet Explorer IE=edge.

La metaetiqueta description de una página proporciona a Google y a otros motores de búsqueda un resumen sobre la página. Google podría utilizarlas como fragmentos de descripción de tus páginas. Se debe de resumir de forma precisa el contenido de la página y utilizar descripciones únicas para cada página.

Se debe de evitar:

  • una metaetiqueta description con contenido no relacionado con la página
  • descripciones genéricas como “Esto es una página web” o “Página sobre cromos de béisbol”
  • una descripción con sólo palabras clave
  • copiar y pegar todo el contenido del documento en una metaetiqueta description
  • utilizar una única metaetiqueta description en todas las páginas de tu sitio o en un gran grupo de páginas de tu sitio

La etiqueta meta viewport

Tamaño de las pantallas de los móviles

Las pantallas de los dispositivos móviles, smartphones y tabletas tiene la particularidad de no tener ventana para las aplicaciones. No tenemos una ventana de navegador.

Para acceder al ancho "físico", real, de las pantallas, está la propiedad width.

En el sitio screensiz podemos consultar las dimensiones de pantalla de algunos smartphones.

La propiedad device-width da acceso al ancho en píxeles que "piensa tener" los smartphones. Ponemos "piensa tener" entre comillas ya que los smartphones que tienen pantallas de alta resolución, com las pantallas Retina de Apple, utilizan valores distintos.

Por ejemplo para los iPhone 5 de pantalla Retina tienen un ancho width de 640 px y ancho device-width de 320 px.

El concepto viewport

Cuando Apple saco el iPhone y su navegador Safari Mobile, introdujo una noción de visualización particular: al abrir una página con Safari Mobile está aparece con 980 px de ancho y un zoom para adaptarse al ancho que supuestamente está disponible device-width, a saber 320 px.
Como consecuencia directa de esto el sitio aparece en miniatura.

Para evitar este efecto desagradable Apple propuso el atributo viewport dentro de la etiqueta meta, que actualmente se ha convertido en un estándar.

La norma es que las dimensiones del viewport sean más pequeñas que las dimensiones reales de la pantalla del dispositivo.

El viewport permite configurar cómo va a interpretar el navegador las dimensiones de la página correspondiente.

El viewport se declara en la cabecera del documento.

      
<meta name="viewport" content="propiedades">

Las propiedades:

  • width: define el ancho que tomará la página del dispositivo. Se puede expresar en px o usar device-width para tomar como anchura la del propio dispositivo.
  • height: definela altura.
  • initial-scale: indica la escala inicial a la que se mostrará la página. Su valor puede ser un número real desde 0.1 en adelante. Un valor de 1 indicará que la web no sufre ningún escalado, se muestra al 100% de su tamaño.
  • minimum-scale y maximum-scale: define los valores máximo y mínimo de escala que se permiten. Será un número real que irá de 0.1 en adelante.
  • user-scalable: define el permiso para hacer o no zoom y escalar el contenido. Acepta los valores yes/no.

una posible declaración puede ser:

<meta name="viewport" content="width=device-width, initial-scale=1, shrink-to-fit=no>

shrink-to-fit=no

Hoy en día, la mayoría de los elementos se diseñan con dimensiones responsivas. Esto significa usar porcentajes y unidades de viewport, en lugar de píxeles simples. Sin embargo, hay algunas circunstancias en las que se incluye un elemento de ancho fijo. Y, el ancho del elemento no es necesariamente más pequeño que el ancho de la ventana gráfica. Por ejemplo, una div de 800px en un iPhone.

Normalmente, el comportamiento que esperamos es que habrá un desbordamiento, y se introducirá una barra de desplazamiento para permitir al usuario ver el resto del ancho del contenido. Sin embargo desde iOS Safari 9.0, cuando Safari está en el nuevo modo de vista dividida (es decir, cuando hay varias aplicaciones abiertas una al lado de otra en la pantalla), esto no sucede.

Lo que sucede en su lugar es que Safari manipula el viewport para que sea igual al ancho del elemento desbordante, p. 800px, por lo que se ajusta en su totalidaden a la pantalla. El efecto de esto es que el resto de la página se "encoge para ajustarse" al elemento más grande.

También podemos ver que cuando el elemento de anchura fija no es mayor que la ventana de visualización, todo es normal.

Según los desarrolladores que trabajan en WebKit, la razón de esto es porque, sobre la base de sus datos, la mayoría de los sitios web hacen un mal uso de la etiqueta meta viewport.

En lugar de utilizar el valor estándar de width = device-width, inicial-scale = 1, tenian tamaños de visor de codificación fija o les faltaban por completo. Por lo tanto, debido a la introducción de Split View, una gran cantidad de sitios web no estaban preparados.

La respuesta de Apple a este problema era, de forma predeterminada, reducir todo el contenido desbordante de la página para ajustarse al ancho de la ventana de visualización del navegador. Para los sitios Web que responden de manera responsable, podemos agregar el nuevo valor meta de la ventana de visualización, shrink-to-fit = no, para señalar esto a Safari y deshabilitar esta característica predeterminada.

<meter>

Define una medida determinada dentro de un contexto de valores especificos.

  • value: el valor del dato en la escala utilizada.
  • min: el mínimo posible.
  • low: el mínimo alcanzado.
  • max: el máximo posible.
  • hight: el máximo alcanzao.
  • optimum: el valor mínimo ideal.
            
<p>Usted ha optenido una nota de 8 sobre 10: <meter value="8" min="0" low="4" max="10" hight="8" optimum="9"></p>
